# Sprint 1 – Sprint Backlog

**Sprint hedefi:** Çalışan temel sistem (üyelik + kayıp ilan + basit arayüz).  
**Süre:** 2 hafta  
**Tarih:** _Başlangıç – Bitiş_

---

## Sprint Backlog öğeleri (Product Backlog’dan seçilenler)

| ID | Görev | Sahip | Durum | Not |
|----|--------|-------|--------|-----|
| 1 | Üyelik: kayıt sayfası (e-posta, şifre, ad) | | Todo / In Progress / Done | |
| 2 | Üyelik: giriş sayfası | | | |
| 3 | Üyelik: çıkış ve oturum kontrolü | | | |
| 4 | Kayıp ilan formu (başlık, açıklama, yer, tarih, kategori) | | | |
| 5 | Kayıp ilanı veritabanına kaydetme | | | |
| 6 | Ana sayfa (basit arayüz) | | | |
| 7 | Giriş yapmamış kullanıcıyı ilan sayfasından yönlendirme | | | |

*Sprint Planning’de bu tabloyu doldurup her göreve “sahip” atayabilirsiniz.*

---

## Kabul kriterleri (Definition of Done)

- [ ] Kullanıcı kayıt olup giriş yapabiliyor.
- [ ] Giriş yapmış kullanıcı “kayıp ilan ekle” sayfasına gidebiliyor.
- [ ] Eklenen kayıp ilan kaydediliyor ve (en azından listeleme olmasa bile) veri tutuluyor.
- [ ] Sprint sonunda çalışan demo gösterilebiliyor.
